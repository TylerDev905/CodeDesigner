Code Designer

A MIPS IDE that can assemble/disassemble Mips32 assembly. It also has support for quickBMS scripts that can be used to unpack and repack file archives.

Conversation Github

Features

  • A powerful assembler that can take mips32 assembly source code and convert it into hexidecimal code.
  • Supports EE, COP0, COP1 and over 285 mips instructions
  • A powerful disassembler that can map mips32 assembly memory dumps/bins.
  • BMS script editor that can pack and unpack archives.
  • Syntax highlighting of all supported languages
  • Code designer syntax that helps cut down the amount of code needed to code in mips assembly
  • Labels that can be shared between the assembler and disassembler windows
  • String dumper that can scan memory dumps or bins

Code Designer Syntax

    • Setreg - sets the register to the 8 digit hex string

      setreg a0, $12345678 or setreg a0, :label

    • Address - sets the address to the 8 digit hex string

      address $12345678

    • Print - will convert the string into its hexidecimal version

      print "hello world"

    • hexcode $12345678

      hexcode $12345678 or hexcode :label

    • call - will pass saved registers into parameter registers and preform a jal instruction

      call $12345678(s0, s1, s2, s3) or call :label(s0, s1, s2, s3)

    Code Designer Labels

    • What is a label?

      A label is a human readable string that will store a reference

    • Assembler labels

      :label using the following syntax in the assembler will tell code designer to store the current address.

    • Disassembler labels

      The 4th column in the disassembler is for storing labels. A label will define the address of the current row

    • Sharing labels between projects

      When an assembler window and a disassembler window are open the labels are automatically shared.

    Disassembler

    • Open a memory dump / .bin

      FIle > Open > select a .bin file

    • String dumper

      The string dumper will search for available strings in the memory dump and allow a person to search through the results. Results will then be displayed in byte formatting in the disassembler.

    • Labels manager

      A code list containing only off codes and labels can be placed in the same folder as the .exe. This file should be named the same as the .bin file opened and contain a .txt extension. Code designer can then try and parse that list and display labels on the addresses given. Code designer will also allow a person to mark an address with labels. These labels will automatically get appended to the off code list and saved.

    • Comments

      Any address can contain a comment. Comments are in the 5th column of the assembler. I tend to use this option when id like to describe the args to a function that has been mapped

    • F9 keypress

      This will change the formatting of an address to a .byte

    • F10 keypress

      This will change the formatting of an address to a .halfword

    • F11 keypress

      This will change the formatting of an address to a .word

    • F12 keypress

      This will set the formatting to use the assembler and syntax highlight the operation
